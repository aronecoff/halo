1. Product & Technology 10 Questions
How does the matching algorithm work?
Halo uses a 6-dimension matching engine that synthesizes behavioral, personality, and contextual data into a compatibility score. The IRIS AI agent conducts a 5-question conversation to build personality profiles, then scores across six dimensions. This multi-dimensional approach is designed to significantly outperform traditional swipe-based systems that rely on limited profile information and aesthetic preference alone. The matching engine is built with tests and CI/CD, deployed to Vercel.
Key Data Points:
• 6-dimension matching engine with IRIS AI personality profiling
• 5-question conversational onboarding via Anthropic Claude
• Working prototype deployed to Vercel (pre-launch, 0 users)
Tone: Technical yet accessible. Lead with "proprietary advantage" language. Emphasize data science rigor.
What's daemon-per-user architecture and why does it matter?
Each user has a dedicated daemon process that continuously learns and personalizes their experience. This architecture enables real-time match scoring, behavioral pattern detection, and instant feedback integration without server-side latency. Unlike traditional shared-server models, daemon-per-user scales linearly and keeps all computation localized, reducing cloud costs while improving response time to sub-100ms latency.
Key Data Points:
• Sub-100ms latency response time
• Linear scaling reduces infrastructure costs
• Localized computation increases data security and user privacy
Tone: Confident in technical differentiation. Position as scalable and cost-efficient infrastructure choice.
How do you handle the cold start problem?
We solve cold start through a structured onboarding that collects behavioral data upfront while users are most engaged. Within the first 15 minutes, we gather sufficient personality, preference, and neural data to generate meaningful matches. Our daemon-per-user system accelerates learning through implicit feedback loops, so even new users see improving match quality within their first week.
Key Data Points:
• 15-minute onboarding generates viable match profiles
• First-week retention metrics validate early engagement
• Implicit feedback loops accelerate personalization
Tone: Solution-oriented. Emphasize rapid value delivery to new users.
What happens when both agents disagree on a match?
When daemon agents disagree on match compatibility, we use a consensus-weighted scoring system that escalates to human review for high-stakes decisions. Disagreement signals often indicate novel patterns worth learning from. We log these instances and use them to retrain our scoring model, treating disagreement as a valuable training signal rather than a failure mode.
Key Data Points:
• Consensus-weighted scoring prevents bad matches
• Disagreement feeds model retraining pipeline
• Human review layer ensures quality on edge cases
Tone: Data-driven and thoughtful. Show how you learn from uncertainty.

We plan to validate match quality through multiple channels: post-match engagement metrics (message volume, response time), relationship outcomes (dating vs. friendship conversion), and direct user feedback surveys. Our next validation milestone (Level 4) involves 10 structured coffee meetings to test match quality in person. We project a 3x+ LTV:CAC ratio by Month 12 post-launch through efficient matching that minimizes wasted user attention.

How does A2A (Agent-to-Agent) negotiation work in matching?
A2A negotiation allows daemon agents to communicate compatibility assessments in real-time. When two users are potential matches, their respective daemons exchange weighted preference vectors and negotiate a consensus score. This enables efficient mutual compatibility checking without exposing full preference data, protecting user privacy while ensuring high-quality bidirectional matches.
Key Data Points:
• Agent negotiation ensures bidirectional matching
• Privacy-preserving through preference vectors (not raw data)
• Real-time consensus scoring for instant feedback
Tone: Sophisticated and privacy-conscious. Highlight technical innovation and user benefit.
What's the feedback loop and how does the system improve over time?
Every interaction—messages sent, responses delayed, relationships formed or ended—feeds back into each user's daemon for real-time model refinement. We batch these signals daily for model retraining, which improves scoring accuracy incrementally. This flywheel effect means match quality compounds: more matches drive more data, which drives better algorithms, which drive higher engagement and retention.
Key Data Points:
• Real-time feedback integration into daemon models
• Daily batch retraining improves algorithm accuracy
• Flywheel: engagement → data → better matching → higher retention
Tone: Confident in continuous improvement. Emphasize virtuous cycle.
